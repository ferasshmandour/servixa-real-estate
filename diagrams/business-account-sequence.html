    Note over User, Notif: ═══ PHASE 1: CREATE BUSINESS ACCOUNT (FR-05) ═══

    User->>+API: POST /api/v1/business-accounts
Headers: Authorization: Bearer {token}
Body: { activity_type_id, city_id,
license_number, name: {ar, en},
activities: {ar, en}, details: {ar, en},
latitude, longitude, files[] }

    API->>API: StoreBusinessAccountRequest
validates all fields

    API->>+Auth: Verify Bearer Token
(auth:api middleware)
    Auth-->>-API: User authenticated ✓

    API->>+BAS: create(validated data)

    BAS->>+DB: INSERT INTO business_accounts
status = 'pending'
    DB-->>-BAS: Business Account #123 created

    BAS->>+DB: INSERT INTO business_account_files
(documents & images)
    DB-->>-BAS: Files saved ✓

    BAS-->>-API: BusinessAccount model

    API-->>-User: 201 Created
{ success: true,
message: "Business account submitted for review",
data: { id: 123, status: "pending", ... } }

    Note over User, Notif: ═══ PHASE 2: VIEW MY BUSINESS ACCOUNTS (FR-07) ═══

    User->>+API: GET /api/v1/business-accounts
Authorization: Bearer {token}

    API->>+BAS: getUserAccounts(user_id)
    BAS->>+DB: SELECT * FROM business_accounts
WHERE user_id = {id}
    DB-->>-BAS: [Account #123 (pending),
Account #99 (approved)]
    BAS-->>-API: Collection of accounts

    API-->>-User: 200 OK
{ success: true, data: [...] }

    Note over User, Notif: ═══ PHASE 3: EDIT BUSINESS ACCOUNT (FR-06) ═══

    User->>+API: PUT /api/v1/business-accounts/123
Body: { name: {ar: "updated", en: "updated"} }

    API->>API: UpdateBusinessAccountRequest
validates + checks ownership

    API->>+BAS: update(123, validated data)
    BAS->>+DB: UPDATE business_accounts
SET name = '...', updated_at = NOW()
WHERE id = 123
    DB-->>-BAS: Updated ✓
    BAS-->>-API: Updated BusinessAccount

    API-->>-User: 200 OK
{ success: true,
message: "Business account updated" }

    Note over User, Notif: ═══ PHASE 4: ADMIN REVIEWS ACCOUNT ═══

    Note over Admin, DB: Admin logs in via /admin/login
(Session auth, guard: admin)

    Admin->>+API: GET /admin/business-accounts
?status=pending
(auth:admin + permission middleware)
    API->>+DB: SELECT * FROM business_accounts
WHERE status = 'pending'
    DB-->>-API: [Account #123, ...]
    API-->>-Admin: Render Blade view
business-accounts/index.blade.php

    Admin->>+API: GET /admin/business-accounts/123
    API->>+DB: SELECT * FROM business_accounts
JOIN business_account_files
JOIN users ON user_id
JOIN cities ON city_id
JOIN activity_types ON activity_type_id
WHERE id = 123
    DB-->>-API: Full account details
    API-->>-Admin: Render Blade view
business-accounts/show.blade.php

    Note over User, Notif: ═══ PHASE 5A: ADMIN APPROVES (FR-08) ═══

    Admin->>+API: POST /admin/business-accounts/123/approve
(requires 'business-accounts.approve' permission)

    API->>+BAS: approve(123)
    BAS->>+DB: UPDATE business_accounts
SET status = 'approved'
WHERE id = 123
    DB-->>-BAS: Updated ✓
    BAS-->>-API: Approved ✓

    API->>+Notif: send(user_id, "Business Account Approved",
"Your account has been approved!")
    Notif->>+DB: INSERT INTO notifications
    DB-->>-Notif: Saved ✓
    Notif->>Notif: Firebase FCM Push
to user's device_token
    Notif-->>-API: Notification sent ✓

    API-->>-Admin: Redirect with success flash
"Account approved successfully"

    Note over User, Notif: ═══ PHASE 5B: ADMIN REJECTS (FR-09) ═══

    Admin->>+API: POST /admin/business-accounts/123/reject
Body: { rejection_reason: "Invalid license" }

    API->>+BAS: reject(123, reason)
    BAS->>+DB: UPDATE business_accounts
SET status = 'rejected',
rejection_reason = 'Invalid license'
WHERE id = 123
    DB-->>-BAS: Updated ✓
    BAS-->>-API: Rejected ✓

    API->>+Notif: send(user_id, "Business Account Rejected",
"Reason: Invalid license")
    Notif->>+DB: INSERT INTO notifications
    DB-->>-Notif: Saved ✓
    Notif->>Notif: Firebase FCM Push
    Notif-->>-API: Notification sent ✓

    API-->>-Admin: Redirect with success flash
"Account rejected"

    Note over User, Notif: ═══ PHASE 6: USER POSTS SERVICE (requires approved account) ═══

    User->>+API: POST /api/v1/services
Body: { business_account_id: 123, ... }

    API->>+BAS: verifyApproved(123, user_id)
    BAS->>+DB: SELECT status FROM business_accounts
WHERE id = 123 AND user_id = {id}
    DB-->>-BAS: status = 'approved' ✓
    BAS-->>-API: Account is approved ✓

    API->>API: Proceed with service creation...

    API-->>-User: 201 Created
{ success: true, data: { service } }

    Note over User, API: ⛔ If account is NOT approved:
    Note over API: 403 Forbidden
{ success: false,
message: "Business account not approved" }
